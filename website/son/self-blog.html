记录我设计这个网站的过程与思考

2026-3-12 起步

为什么要做这个网站？

刚开始学习 HTML 和 CSS 的时候，老师布置了一个作业：做一个个人网站。 一开始觉得挺难的，不知道从哪里下手。后来决定先把结构想清楚， 再一页一页地写，慢慢就做出来了。

这个网站包含了首页、登录、注册、简历、照片墙、博客和关于页面， 涵盖了大部分 HTML 基础标签的使用场景，算是一次比较完整的练习。

2026-3-13 布局

Flex 布局让我少走了很多弯路

在做首页四栏卡片的时候，我一开始用 float 来排列， 结果出现了各种奇怪的问题，元素塌陷、对不齐…… 后来改用 Flex 弹性布局，只需要三行代码就解决了：

display: flex;

gap: 20px;

flex: 1;

Flex 是目前最主流的 CSS 布局方式之一，几乎所有现代网站都在用。 另一个主流方案是 Grid 网格布局，适合更复杂的二维排列， 是我下一步想学习的内容。

2026-3-15 交互

用 CSS 就能做交互，不一定需要 JavaScript

做简历页的时候，我想实现"鼠标放上去才显示内容"的效果。 本以为需要写 JavaScript，后来发现只用 CSS 的 :hover 伪类 配合 opacity 就能实现，而且还带了淡入动画：

.card-content { opacity: 0; transition: opacity 0.3s; }

div:hover .card-content { opacity: 1; }

这让我意识到，CSS 其实比想象中强大很多。 现代 CSS 还支持自定义变量、动画关键帧（@keyframes）、媒体查询等高级特性， 有很大的探索空间。

2026-3-18 技术展望

接下来想学的技术

做完这个网站，我列了一个接下来想学习的技术清单：

  • JavaScript：让页面真正"动"起来，实现更复杂的交互逻辑
  • Vue / React：目前最主流的前端框架，适合开发大型单页应用
  • Node.js：用 JavaScript 写后端，实现登录注册的真实数据存储
  • MySQL：数据库基础，配合后端存储用户信息
  • Git：版本控制工具，管理代码历史，方便团队协作
  • 响应式设计：让网站在手机和平板上也好看，深入学习媒体查询
2026-3-20 总结

一周学 CSS 的感受

学了一周 CSS，最大的感受是：动手做比反复看教程有用得多。 很多属性光看文字描述根本记不住，但是自己改一改数值、看看效果， 马上就明白是怎么回事了。

推荐几个学习过程中用到的工具和网站：

  • MDN Web Docs：最权威的 HTML / CSS 参考文档
  • VS Code：写代码用的编辑器，配合 Live Server 插件实时预览
  • 浏览器开发者工具（F12）：调试 CSS 必备，可以直接在浏览器里改样式
  • Can I Use：查某个 CSS 属性的浏览器兼容性
